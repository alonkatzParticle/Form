# Particle for Men — Static Creative Ticket Generation System

Generate production-ready Monday.com creative tickets for Particle for Men static image ads. This system encodes the complete ticket creation intelligence extracted from 752 production tickets, 322 concept briefs, 3,287 update threads, 108 supporting text fields, 820 correction/feedback exchanges, and 18+ months of the Design Projects 2.0 Monday board. Every field format, naming convention, concept category, versioning rule, brief structure, slogan, review, comparison copy, long-read body, and correction pattern is reverse-engineered from what the team actually shipped.

This skill covers STATIC IMAGE ADS ONLY — not video scripts, hooks, or ad copy (those are handled by the `particle-creative-tickets` skill).

## CRITICAL FIRST STEP: Read the Copy Database

Before generating ANY ticket that requires slogans, reviews, headlines, comparison copy, or long-read text, read the reference file at `references/copy-database.md`. This file contains the COMPLETE production-proven copy arsenal for every product — 500+ slogans, 30+ reviews, 4 full long-read bodies, 50+ comparison structures, all Hebrew/French/Spanish/Italian translations, and every GT article headline ever used. Use this copy as the foundation — never invent slogans from scratch when proven copy exists.

## SECTION 1: TICKET ANATOMY — The Monday Board Schema

### 1.1 Ticket Naming Convention (Mandatory Pipe-Delimited Format)

Every ticket name follows this exact structure, extracted from 752 production tickets:

```
[Product] | [Platform] | [Source/Tool (optional)] | [Concept Title]
```

**Segment rules:**

| Segment | Required | Rules | Examples |
|---------|----------|-------|----------|
| **Product** | YES | Exact product name from the canonical product list (Section 5). Use the official short name, not abbreviations. For multi-product ads, use `Multiple Products`. | `Face Cream`, `Anti-Gray Serum`, `Ab Firming Cream` |
| **Platform** | YES | `META` or `Meta` (interchangeable in production). Use `GT` prefix within the source segment for Gentleman Today article ads. | `META`, `Meta` |
| **Source/Tool** | NO | Include ONLY if the concept was generated by an AI tool or is for Gentleman Today. Valid values: `ADGE`, `GT`, `ChatGPT`, `Upspring`, `MOTION`. Can be combined: `GT \| ChatGPT`. | `ADGE`, `GT` |
| **Concept Title** | YES | Descriptive, specific. Often includes the hook, slogan, or visual concept name. Should be unique enough to identify the ad at a glance. | `Before & After AI 45+ men (slogan)`, `Empty Shelf`, `Ditch the Hair Dye Idea` |

**Suffix conventions used in production:**
- `(copy)` — iteration of an existing ticket with modified copy only, same visual concept
- `(French)` / `(Spanish)` / `(Italian)` / `(Hebrew)` — localized version of an existing ticket
- `#2`, `#3`, `#4` or `v2`, `V2` — sequential iteration (new batch of same concept)
- `(no background, no stamp)` — variant removing specific visual elements
- `(slogan)` — B&A images that include slogan text overlay
- `(new creators)` — same concept but with fresh creator images

| **Ad Name** | NO | — | Final Meta ad name, rarely filled at ticket creation |

### 1.3 Sizing Specification Rules

From 752 tickets — these are the empirical size distributions:

| Frequency | Size Value | Pixel Dimensions | When to Use |
|-----------|-----------|-----------------|-------------|
| **82%** (613) | `1x1, 9x16` | 1200×1200 + 900×1600 | **DEFAULT for all static image ads** |
| **8%** (61) | `4x5, 9x16` | 1080×1350 + 900×1600 | Long-read statics, comparison tables, content-heavy layouts requiring extra vertical space |
| **4%** (30) | `9x16, 4x5` | Same as above | Order-reversed variant (no functional difference) |
| **2%** (16) | `9x16, 1x1` | Same as default reversed | No functional difference |
| **1%** (9) | `1x1` | 1200×1200 only | Square-only (rare, product shots for catalog) |
| **<1%** | `16x9, 1x1` | 1200×627 + 1200×1200 | Landscape + square (very rare) |

**Decision rule:** Default to `1x1, 9x16`. Switch to `4x5, 9x16` only when the concept needs extra vertical canvas: long-read copy, multi-row comparison tables, multi-section layouts, or content-heavy ad formats.

**Always state total image count in the brief:** `"Create [N] images in 1200×1200 and 900×1600 px (total: [N×2] images)"`

### 1.4 Version Count Decision Tree

| Concept Category | Default Versions | Reasoning |
|-----------------|-----------------|-----------|
| Standard slogan + product | 3 | Different backgrounds, product angles, or layouts |
| Before & After (AI-generated) | 5–10 | AI variance requires volume for quality control |
| Before & After + slogans | 6 | Each slogan paired with unique B&A image |
| Sale/promotional banners | 4–5 | Multiple design styles per product |
| Comparison / split-screen | 2–3 | Each version = different copy comparison set |
| Testimonial / review | 3–5 | Each version = different review text |
| Long-read static | 1–2 | Each version = different product visual, same body copy |
| GT article ads | 5 | Multiple native-style images for testing at scale |
| Inappropriate billboard | 3 | Each = different visual execution of same slogan |
| Remarketing / retargeting | 4 | Different product shots with identical offer text |
| Creator + slogan | 3–6 | Different creators, slogans, or creator×slogan combinations |
| Product shot + text | 3–5 | Different angles, backgrounds, text treatments |
| Chat/text message format | 2–3 | Different conversations/scenarios |
| Animated / GIF | 2 | Default deliverable spec: "2 versions, 10-second loop" |
| Catalog / dynamic | Varies | One per product in the catalog set |
| Sale (localized) | Same as English | Mirror the English version count |
| Google Search screenshot | 3–5 | Different search queries or result layouts |

**Total images formula:** `Versions × Number of Size Combos = Total Images`
Always calculate and state this in the brief: "5 versions in 1x1, 9x16 = 10 images total"


## SECTION 3: CONCEPT BRIEF WRITING FORMULA

### 3.1 Brief Anatomy (4 Required Components)

Every concept brief must include these four components. Briefs average 414 characters (median 318). Be specific but concise.

**Component 1 — VISUAL DIRECTION** (always first)
```
What the image should look like:
- Scene/setting: bathroom, gym, studio, drugstore shelf, bedroom, outdoor
- Subject: man 45+, couple 50+, woman, hands close-up, product alone
- Composition: split-screen, close-up, full-body, product hero, grid/collage
- Background: dark/aggressive, clean/minimal, lifestyle, blurred, brand gradient
- Product placement: hero center, corner insert, held by creator, on counter/nightstand
- Lighting/mood: premium, natural/UGC, dramatic, cinematic
```

**Component 2 — TEXT / COPY ON IMAGE**
```
All text that appears on the finished image:
- Main headline/slogan (with numbered alternatives per version)
- Supporting text / subheadline
- CTA text: "Shop Now" / "Visit ParticleForMen.com"
- Trust badges: "30-Day Money-Back Guarantee" / "Free Shipping" / "1M+ Customers"
- Review stars: ★★★★★
- Discount: "[X]% Off" / "Use Code [CODE]"
```

**Component 3 — VERSIONING INSTRUCTIONS**
```
What changes between versions (must be explicit):
- Different backgrounds (dark vs light, color A vs color B)
- Different creators/models (link to asset folder)
- Different slogans (each paired with a version number)
- Different product angles (front, 45°, lifestyle)
- Different layouts (text left vs right, horizontal vs vertical split)
```

**Component 4 — TECHNICAL NOTES**
```
- Specific constraints: "no text, no product" / "no crazy abs" / "realistic only"
- Reference files: "see attached" / "see example" / [Dropbox link]
- Quality directive: "make as attention-grabbing as possible"
- Adaptation notes: "try to make the imagery as attention-grabbing as possible,
  inspired by the 2 attached top-performing images but not too similar"
```

### 3.2 Brief Length Calibration

| Concept Complexity | Target Brief Length | Examples |
|-------------------|-------------------|---------|
| Simple (product + slogan) | 50-150 chars | "Slogan image with the product" |
| Standard (B&A, comparison, testimonial) | 200-400 chars | Most production tickets |
| Complex (ADGE, long-read, multi-version) | 400-800 chars | Elaborate visual scenes, multiple versions |
| Sale template (batch) | 300-500 chars | Standardized with design references |

### 3.3 Brief Writing Anti-Patterns

| Anti-Pattern | Why It Fails | Correct Approach |
|-------------|-------------|-----------------|
| Vague visual direction ("make it look nice") | Designer interprets freely → revision cycles | Specify scene, lighting, composition, mood |
| No version differentiation ("create 5 versions") | Designer produces near-identical variants | State what changes per version: background, copy, layout |
| Missing product placement instruction | Designer guesses product location | State: "product in corner" / "hero shot center" / "held by creator" |
| Copy without alternatives | One slogan = limited testing | Always provide 2-5 slogan alternatives |
| No reference file or link | Designer starts from scratch | Always attach reference or link to similar successful ad |
| Unrealistic B&A transformations | Gets flagged by Meta, loses audience trust | Specify realistic constraints ("no crazy abs", "subtle improvement") |
| Mixing multiple concepts in one ticket | Confuses scope, slows production | One ticket = one concept. Create separate tickets for distinct concepts |
| Forgetting total image count | Designer delivers wrong quantity | Always state: "total: [N] images" |

**Step 1: Identify Requirements**
Ask or determine:
- Product (which product/bundle?)
- Concept category (which of the 20 categories?)
- Special requirements (sale event? localization? iteration of existing concept?)

**Step 2: Select Category Template**
Match to the appropriate concept category from Section 2. Use the exact brief template for that category.

**Step 3: Generate Ticket Name**
Apply the naming convention from 1.1. Verify it follows the pipe-delimited format and matches production patterns.

**Step 4: Populate Metadata**
Fill all required fields from 1.2 with appropriate defaults:
- Priority: `Medium` (unless urgent → `High` or `Critical ⚠️️`)
- Status: `Ready For Assignment`
- Sizes: `1x1, 9x16` (unless category requires `4x5, 9x16`)
- Versions: Use decision tree from 1.4
- Platform: `Meta` (default)

**Step 5: Write Concept Brief**
Follow the 4-component structure from 3.1:
1. Visual direction (specific scene, subject, composition, background, product placement)
2. Text/copy on image (headline alternatives, supporting text, badges, CTA)
3. Versioning instructions (what changes between versions)
4. Technical notes (constraints, references, quality directives)

**Step 6: Write Supporting Text**
If the concept includes slogans, reviews, or copy variations, populate the Supporting Text field separately from the Concept field.

**Step 7: Quality Check**
Before presenting the ticket, verify:
- [ ] Name follows pipe-delimited format with correct product name
- [ ] Sizes match the concept complexity (1x1,9x16 default; 4x5,9x16 for dense content)
- [ ] Version count is appropriate for the concept category
- [ ] Concept brief includes all 4 components (visual, text, versioning, notes)
- [ ] Supporting text is provided if slogans/reviews are referenced
- [ ] Total image count is stated ("total: [N] images")
- [ ] No unrealistic transformation claims in B&A concepts
- [ ] Slogans are under 12 words each and attack different angles
- [ ] Product name matches canonical list exactly

**Step 8: Present Ticket**
Format the ticket as a clean, copy-pasteable Monday board entry with all fields clearly labeled.

## SECTION 10: CORRECTION INTELLIGENCE — Patterns From 820 Feedback Exchanges

Analysis of 820 correction/feedback exchanges in the update threads reveals recurring patterns. Encode these into briefs to PREVENT revision cycles.

### 10.1 Copy Corrections (Most Frequent)

| Pattern | Example | Prevention |
|---------|---------|------------|
| **Typo in slogan** | "result" → "results" (AGS ticket) | Proofread every slogan. Plurals matter. |
| **Unnecessary CTA text** | "Let's just remove the part about 'Get Yours...'" (AFC) | Keep CTA separate from headline. Don't stack CTAs. |

## SECTION 14: ADVANCED TICKET PATTERNS

### 14.1 The "(copy)" Iteration Pattern

When a visual concept performs well and needs new copy variations:
1. Keep the original ticket name
2. Append `(copy)` suffix
3. In the Concept field, reference the original ticket: "Same visual as [Original Ticket Name], new copy/slogans."
4. Keep same sizes and version count (or adjust based on new copy count)
5. Change only the Supporting Text with new slogans/headlines

### 14.2 The "v2" / "#2" Iteration Pattern

When a concept type needs a fresh batch (new images, same approach):
1. Append `v2` or `#2` to the concept title portion
2. In the Concept field: "Create [N] MORE variations of this concept" or reference the original with "same style but different [creators/backgrounds/slogans]"
3. Often includes instruction: "inspired by the attached top-performing images but not too similar"

### 14.3 The Localization Pattern

For translating existing English tickets to other languages:
1. Create a NEW ticket with language suffix: `[Original Name] (French)` or `in French`
2. Reference the English ticket by Monday board URL: `https://particle-for-men.monday.com/boards/8036329818/pulses/[PULSE_ID]`
3. Concept: "Use the design you created for the English versions... to create versions in [Language] by just changing the text to: [translated text]"
4. Keep identical sizes and version count

### 14.4 The Multi-Product Sale Batch

When a sale event launches, create tickets in this order:
1. **Core products first** (FC, AGS, AFC, IM, SHG, Gravité, Shampoo, NC, Deodorant) = 9 tickets
2. **Extended products** (HC, Varros, Sunscreen, BW, HRK, Scar Gel) = 6 tickets
3. **Bundles** (LKK, GB, PSS, SSS, HGB, UGB) = 6 tickets
4. **Multi-product homepage** = 1 ticket
5. **Localized versions** (FC Hebrew, FC French, AGS French, AGS Italian) = 4+ tickets

Total per major sale: **26-30 tickets**

### 14.5 The Template Reference Pattern

For mature sale events (BFCM, Easter, New Year's), the team uses established templates:
- Brief says: "using the [Sale Name] template" (no further design direction needed)
- This is only valid AFTER the first sale of that type has been designed
- For the FIRST instance of a new sale, provide full design direction referencing prior sales: "in the style of our top performers from [previous sale]"

### Gate 2: Metadata Completeness
- [ ] All 7 required fields populated (Name, Priority, Status, Product/Bundle, Department, Sizes, Versions)
- [ ] Priority is appropriate (Medium default, High for urgent, Critical for blocking)

- [ ] Visual direction specifies: scene, subject, composition, background, product placement
- [ ] Text/copy lists all text that appears on the image with numbered alternatives
- [ ] Versioning instructions specify what changes between versions
- [ ] Total image count stated: "total: [N] images"
- [ ] Reference files linked if applicable
- [ ] Realism constraints included for AI B&A concepts

### Gate 4: Copy Quality
- [ ] Slogans ≤12 words each
- [ ] Each slogan attacks a DIFFERENT angle
- [ ] No generic superlatives ("amazing", "incredible", "wonderful")
- [ ] Masculine voice maintained (no "skincare routine" without reframing)

- [ ] If comparison ad: competitor brand blurred or genericized

- [ ] If sale: discount code matches seasonal calendar
